# p1 — 수치 적분의 수렴성 (Numerical Integration Convergence)

이 섹션에서는 물리 시뮬레이션에서 자주 사용하는 단순한 업데이트 규칙이
**dt → 0 일 때 연속적인 물리 해에 수렴한다는 사실**을 설명한다.

대표적인 시뮬레이션 업데이트는 다음과 같다.

```
x += v * dt
v += a * dt
```

이 단순한 규칙은 시간 간격이 충분히 작아지면 **연속적인 물리 방정식을 근사하게 된다.**

이 섹션은 세 가지 경우를 설명한다.

---

## Case 1 — 속도에 비례하는 저항력 (Drag Force)

저항력이 속도에 비례한다고 가정하자.

```
F = -k v
```

뉴턴의 법칙에 의해

```
m dv/dt = -k v
```

따라서

```
dv/dt = -(k/m) v
```

해석적 해는 다음과 같다.

```
v(t) = v0 e^{-(k/m)t}
```

---

### 시뮬레이션 업데이트

시뮬레이션에서는 다음 규칙을 사용한다.

```
v += -(k/m) v * dt
```

이는 다음과 같다.

```
v_{n+1} = v_n (1 - (k/m) dt)
```

따라서

```
v_n = v0 (1 - (k/m) dt)^n
```

시간이

```
t = n dt
```

일 때

```
v(t) = v0 (1 - (k/m) dt)^{t/dt}
```

e의 정의를 이용하면

```
lim_{dt→0} (1 - a dt)^{t/dt} = e^{-at}
```

따라서

```
v(t) → v0 e^{-(k/m)t}
```

즉, 시뮬레이션 결과는 **dt → 0 일 때 실제 해로 수렴한다.**

---

## Case 2 — 속도를 적분하여 위치 구하기

속도 함수 (v(t))가 주어졌다고 가정하자.

미분 방정식은

```
dx/dt = v(t)
```

이다.

따라서 위치는

```
x(t) = x(0) + ∫ v(t) dt
```

---

### 시뮬레이션 업데이트

시뮬레이션에서는 다음 연산을 반복한다.

```
x += v * dt
```

이는 다음과 같다.

```
x_{n+1} = x_n + v(t_n) dt
```

반복하면

```
x_n = x_0 + Σ v(t_k) dt
```

이 식은 정확히 **적분의 리만 합 (Riemann Sum)** 이다.

따라서

```
dt → 0
```

일 때

```
x(t) = x_0 + ∫ v(t) dt
```

로 수렴한다.

---

## Case 3 — 가속도 적분

가속도 (a(t))가 주어졌다고 가정한다.

연속 방정식은 다음과 같다.

```
dv/dt = a(t)
dx/dt = v(t)
```

---

### 시뮬레이션 업데이트

시뮬레이션은 다음과 같이 수행된다.

```
v += a * dt
x += v * dt
```

속도 업데이트는

```
v_{n+1} = v_n + a(t_n) dt
```

이므로

```
v(t) = v_0 + ∫ a(t) dt
```

를 근사한다.

그리고

```
x_{n+1} = x_n + v_{n+1} dt
```

은

```
x(t) = x_0 + ∫ v(t) dt
```

을 근사한다.

따라서

```
a → v → x
```

과정 전체가

```
dt → 0
```

일 때 실제 물리 궤적으로 수렴한다.

---

## 원운동 시뮬레이션 (Circular Motion)

기반 코드:

```
p3_circle.cpp
```

원운동은 다음 방정식을 따른다.

```
a = -ω² x
```

---

### 업데이트 순서

시뮬레이션에서는 다음 순서를 사용한다.
단, 초기 속도는 위치보다 0.5 dt만큼 더 변화해 놓은 걸 써야 한다.

```
v += a * dt
x += v * dt
```

이 방식은 **Semi-Implicit Euler Method**에 해당한다.

속도를 먼저 업데이트하면 수치적으로 더 안정적인 결과를 얻을 수 있다.

반대로

```
x += v * dt
v += a * dt
```

순서를 사용하면 오래된 속도를 사용하게 되어 오차가 커질 수 있다.

---

## 격자 기반 열 전도 시뮬레이션 (Heat Conduction)

기반 코드:

```
p3_grid.cpp
```

열 전도는 다음 방정식을 따른다.

```
∂T/∂t = α ∂²T/∂x²
```

---

### 격자 이산화

공간을 격자로 나눈다.

```
T[i][j]
```

공간 2차 미분은 다음과 같이 근사한다.

```
∂²T/∂x² ≈ (T[i+1] - 2T[i] + T[i-1]) / h²
```

따라서 업데이트 규칙은

```
T_i^{new}
=
T_i + α dt (T_{i+1} - 2T_i + T_{i-1}) / h²
```

이 된다.

이를 통해 열이 인접한 격자 셀로 확산되는 과정을 시뮬레이션할 수 있다.

# p5 — 테일러 급수 시뮬레이션

이 섹션에서는 **테일러 급수 (Taylor Series)** 를 이용한 함수 근사를 설명한다.

예를 들어 지수 함수는 다음과 같이 전개된다.

```
e^x = Σ (x^n / n!)
```

---

### 시뮬레이션

프로그램은 다음 값을 계산한다.

```
S_N(x) = Σ x^n / n!
```

항의 개수 (N)을 증가시키면 근사값은 실제 값 (e^x)에 가까워진다.

---

### 수렴성

지수 함수의 테일러 급수는 모든 실수에서 수렴한다.

```
lim_{N→∞} S_N(x) = e^x
```

시뮬레이션은 항의 개수가 증가할수록 근사가 개선되는 과정을 보여준다.
단, 테일러 급수의 수렴성은 균등수렴이 아닌 점별수렴이라 N을 무한히 키운다고 아무 x에 대해 함수가 수렴함을 보장하지 않는다. 따라서 테일러 급수는 국소적 영역에서 함수를 근사하기에 더 적합하다.

---

# 부록 — 수치 미분 (Numerical Differentiation)

수치 미분은 함수 값을 이용하여 도함수를 근사하는 방법이다.

전진 차분:

```
f'(x) ≈ (f(x+h) - f(x)) / h
```

중앙 차분:

```
f'(x) ≈ (f(x+h) - f(x-h)) / (2h)
```

중앙 차분은 일반적으로 더 작은 오차를 가진다.

---
